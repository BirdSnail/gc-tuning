# GC调优实战：减少Full GC的次数

在我们的一个应用中，由于场景特殊（需要在内存开辟较大的临时空间用于计算），
应用会出现频繁的老年代GC和Full GC。请根据具体的业务场景调整[这里](https://github.com/hcsp/gc-tuning/blob/master/src/test/java/com/github/hcsp/GcTest.java)的JVM启动参数，
使得测试中老年代GC/Full GC出现的次数小于3次。
